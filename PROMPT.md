# 要求仕様: 二重振り子シミュレーション

## 概要

カオス的な動きを見せる二重振り子をブラウザ上でシミュレートする物理演算Webアプリを作成

## 目的

- 単純なルールから生まれる複雑な動きを視覚化
- 初期条件の微小な違いが大きな差を生む「カオス」を体感
- AIコーディングで物理シミュレーションを短時間で作成するデモ

## 機能要件

### 必須機能

1. 二重振り子の描画
   - 2つの振り子（腕と重り）を連結して表示
   - 滑らかなアニメーション（60fps目標）

2. 物理演算
   - ラグランジュ力学に基づく運動方程式
   - 重力加速度の適用
   - リアルタイム計算

3. 軌跡の描画
   - 先端の重りの軌跡を線で表示
   - 時間経過でフェードアウト（または一定長さで消去）

### 追加機能（実装優先度順）

1. パラメータ調整UI
   - 腕の長さ（L1, L2）
   - 重りの質量（m1, m2）
   - 重力加速度（g）
   - 初期角度（θ1, θ2）

2. 複数振り子の同時表示
   - 初期条件をわずかに変えた複数の振り子を同時に表示
   - カオス的な発散を視覚的に比較

3. 再生コントロール
   - 一時停止 / 再開
   - リセット
   - スロー再生

4. エネルギー表示
   - 運動エネルギー / 位置エネルギー / 全エネルギーをグラフ表示

## 技術要件

### フロントエンド

- Canvas 2D（シンプルで十分）
- バニラJS

### 物理計算

以下のいずれかのアプローチを検討:

1. **自前実装（推奨度: 中）**
   - ラグランジュ力学に基づく運動方程式
   - Runge-Kutta法（4次）で数値積分
   - フレームレートに依存しない固定タイムステップ
   - 注意: 実装が複雑で、数値誤差による不自然な挙動が発生しやすい

2. **物理エンジン使用（推奨度: 高）**
   - **Matter.js**: 制約（Constraint）で2つの振り子を接続。安定性が高い
   - **Planck.js**: Box2Dベースで高精度。複雑な物理も正確に再現
   - メリット: 物理の正確性をエンジンに任せられる
   - デメリット: ライブラリの依存が増える

```javascript
// Matter.js での実装例
const pivot = { x: 400, y: 100 };
const bob1 = Matter.Bodies.circle(400, 200, 20);
const bob2 = Matter.Bodies.circle(400, 300, 20);

Matter.Composite.add(world, [
  Matter.Constraint.create({ pointA: pivot, bodyB: bob1, length: 100 }),
  Matter.Constraint.create({ bodyA: bob1, bodyB: bob2, length: 100 })
]);
```

物理エンジンを使う場合は、見た目の調整（軌跡描画、色など）に集中できる

### パフォーマンス

- 60fps を目標
- 軌跡の描画点数を制限してメモリ節約

## UI/UXガイドライン

- ダークテーマ（軌跡が映える）
- 振り子は明るい色、軌跡はグラデーション
- パラメータUIは画面端に配置
- レスポンシブ対応（縦長画面でも適切に表示）
